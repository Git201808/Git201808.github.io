hexo使用中的一些问题

发表于 2021-09-05 更新于 2021-09-06

1. 端口

在使用hexo布署到github时会提示端口被占用，一般来说，出现这种情况，要么是被别的应用程序给占了，要么是打开了本地预览窗口没关导致的。

1.1 被别的程序占用端口

排查是什么占用的，能找到的，具体情况具体分析。能不能改，不能改就去改hexo的配置，详情参考Hexo文档

1.2 已打开的本地预览，占用端口

这个有点蠢没得说的，我也遇到了。直接关了就是

2. 博客中发现博客没有标题

初次使用时，急急忙忙就部署了，刷新一看，昨没有标题呀。明明我有写一级标题的，原因就是部署的时候没输一个命令，当时完全无视了hexo clean这个命令，清理什么的。解决办法，重输以下命令，刷新即可

3. 图片没上传

使用md编写的格式，原文件中有图片信息，图片信息引用至typeof的目录下，部署后发现没有图片。

安装使用插件

4. 新写的博客部署不上

发现有些新写的博客，没有更新到github上，

原因：使用hexo new 'fisrt-time'时生成的md文件中，有标题有时间，还有个待填写的tags，原因是tags:java贴得太近了。会导致部署的时候，该博客没有更新上

ssh远程到github

发表于 2021-09-05 更新于 2021-09-06

github是我们经常使用的工具，需要经常链接到github中去，每次网页操作就显得麻烦得很，这时使用ssh密钥，就可以解决这个问题。

1.生成密钥

使用ssh -keygen命令生成本地密钥,使用git的bash进行操作

2.添加到github中

进入github网站，国内比较慢，需要自行解决，进入后找到设置项，在设置中找到ssh&GPG keys，在ssh中点击ssh keys –> New SSH key,将本地的ssh -keygen生成的id_rsa.pub文件中的密钥信息添加到ssh key

3.测试
